Technical Specification
Neural Child.
In detail.
5
Interacting Networks
The architecture consists of five distinct neural networks that communicate with each other. No single network owns the model. Behavior emerges from interaction.
Emotional
Memory System
Context-weighted recall. The emotional state at encoding influences retrieval priority. Memory is not neutral storage.
Dev. Stages
Growth Model
The architecture includes developmental stage progression. The model does not start as a fully formed adult intelligence. It grows.
PyTorch
Framework
Deep learning substrate. Local training on research hardware. No cloud dependency for the core research loop.
Research
Stage
Pure research. No commercial deployment planned. No product roadmap. The question is the point.
2019–
Ongoing
Neural Child has been an active research question since Christopher began systematic self-experimentation. The architecture formalizes intuitions that were years in the making.
What if intelligence grows
instead of being deployed?

Neural Child began with a question about developmental AI: what changes if the architecture models growth stages rather than starting fully capable? The emotional memory system emerged from that question. The five-network interaction followed.

Build History
The record.
Curiosity
The original question.
What does it mean for an AI to develop rather than be deployed? Not a product question. A research question. The commercial angle was explicitly excluded from the start.
Architecture
Five networks, not one.
The decision to use five interacting networks rather than a single unified model. Each network owns a domain. Behavior is emergent from the interactions between them.
Emotional Memory
Context-weighted recall.
The emotional memory system: encoding state influences retrieval priority. Memory is not neutral storage in this architecture.
