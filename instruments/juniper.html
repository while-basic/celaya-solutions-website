Technical Specification
Juniper.
In detail.
7
OpenClaw Agents
Executive, Vitals, Ledger, Bridges, Sentinel, Ops, Muse. Each agent owns one domain. The OpenClaw architecture uses a centralized executive agent with six specialized subordinates.
9
Active Cron Jobs
Nine scheduled tasks run continuously. Each cron job is assigned to a specific agent and a specific cadence. No manual triggering required for routine work.
Local
Ollama Models
All cron jobs run on local Ollama models. No API calls for scheduled tasks. The cloud is reserved for situations requiring attention and judgment.
Escalation
Cloud Only
When a cron job encounters something that requires attention, it escalates to Claude Sonnet or Opus. This keeps Anthropic API quota consumption minimal and intentional.
Deployed
24/7 Active
All 9 cron jobs are running. Juniper does not have office hours. The agents execute on schedule regardless of whether the researcher is actively working.
0
Quota Burn
Minimized Anthropic API usage by design. Local Ollama models handle the volume. Cloud models handle the exceptions. The architecture is economical by principle.
The routine work
runs locally.

Juniper is built around a core principle: most routine cognitive work does not require the most capable model. Local Ollama models are sufficient for scheduling, logging, and monitoring. Cloud models engage only when the work warrants it.

Build History
The record.
Architecture
OpenClaw defined.
The 7-agent OpenClaw architecture: one executive agent that coordinates six specialized agents. The executive is the only agent with full context. Specialists report upward.
Local First
Ollama integration.
The decision to run all cron jobs on local Ollama models. Not a cost optimization — a design principle. Routine work should not require internet access or API quotas.
9 Crons
Scheduled automation.
Nine cron jobs assigned across the 7 agents. Each job has a cadence, a model assignment, and an escalation threshold. Fully specified before deployment.
Now
Running continuously.
All 9 jobs active. Juniper monitors, logs, bridges, and reports. When something requires the researcher's attention, it surfaces it. Everything else runs quietly.
